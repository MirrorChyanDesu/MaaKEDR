# 连接设置

MaaKEDR 通过 ADB 连接模拟器或 Android 设备。正确配置连接是任务执行的前提。

---

## 自动检测

大多数情况下，保持有且仅有目标模拟器启动，MaaKEDR 的 **连接** 板块会自动检测到它。点击 **刷新** 可重新扫描。

检测到模拟器后，**连接状态** 指示灯变为绿色即可开始使用。

---

## 手动配置

如果自动检测失败，可点击 **自定义** 手动配置：

1. **ADB 路径**：通常自动填充，一般无需修改
2. **ADB 地址**：`<IP>:<端口>` 格式，如 `127.0.0.1:16384`

各模拟器默认 ADB 端口：

| 模拟器         | ADB 地址          | 备注                    |
| :------------- | :---------------- | :---------------------- |
| MuMu 模拟器 12 | `127.0.0.1:16384` | 需在设置中开启 ADB 调试 |
| BlueStacks 5   | `127.0.0.1:5555`  | 需开启 ADB 调试         |
| LDPlayer 9     | `127.0.0.1:5555`  | 默认已开启              |
| 夜神模拟器     | `127.0.0.1:62001` | 默认已开启              |

> [!TIP]
>
> 可在模拟器设置中查看 ADB 调试开关。MuMu 12 需在"设置中心 → 其他"中开启 ADB；BlueStacks 需在"设置 → 高级 → Android 调试桥"中开启。

---

## 连接验证

在终端中执行以下命令可手动验证 ADB 连接：

```shell
adb devices
```

正常输出应显示已连接的设备列表，状态为 `device`。

---

## 多模拟器实例

如果启动了多个模拟器实例，MaaKEDR 会列出所有检测到的设备。请确保选择正确的目标实例。建议运行 MaaKEDR 时只启动一个模拟器实例以避免混淆。
